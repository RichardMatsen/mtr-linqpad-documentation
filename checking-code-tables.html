      # Checking Batch-Load Spreadsheets

      # Background

      Code tables (Reference or Master Data) are static data tables such as:

      • code+description lookup tables (e.g Title, Gender)
      • core model entities such as Consultants, Clinical Specialties (Departments)
      • statutory codes for hospital activities (Procedures, Diagnoses)
      • diary records such as Outpatient Clinics schedules

      # Code tables vs Migration Data

      Migrated data is the core patient data plus all related transactional data, such as appointments, hospital stays, A&E visits, medical notes.

      When a trial migration is run, the code tables are expected to be in place and complete. In practice, this is never the case since the old PAS system is still operational, Master data will change over time (Consultants come and go, Clinic definitions change as hospital focus changes).

      # Bulk Loading

      To help analysts in gathering Master data, a set of Excel workbooks are provided. Before the records can be loaded, they require validation for mandatory fields, unique values across the set, and referential integrity for foreign keys to other tables.

      VBA macros in the workbook provide this to a degree, but not RI checks between other workbooks and data already loaded.

      In addition, as the workbooks grew larger performance degraded to the point where analysts stopped using the validation macros altogether.

      # Linqpad queries to provide bulk validation

      To provide faster and more complete validation, a set of Linq queries were built in Linqpad.

      # Features

      • Access to Excel, CSV, and database records, provides queries across these disparate sources.
      • Normalize function for matching codes with ignore-case, non-alpha character stripping
      • Central definition of validations rules, easy to modify and adapt.

      # Query code

      // Check batch file for loading Wards code table
// -----------------------------------------------------------------------
// Fetch incoming batch data
var folder = @"F:\Extracts\";
var batch_parser = r=> new {
  Code= r[0],
  Desc= r[1],
  StartDate= r[3],
  EndDate= r[3],
  Type= r[4]
};
var batch = FetchFromCodeTableBatch("Wards.csv", batch_parser, folder);

// Fetch wards already loaded
var query = "SELECT Ward_Code, Ward_Desc FROM Wards";
var parser = r=> new { Code = r[0], Desc = r[1] };
var wards = FetchFromDatabase(query, parser);

// Validation 1: Duplicate descriptions
var dup_desc = from c in batch
  group c by c.Desc.Normalize() into g
  where g.Count() > 1
  select g;
dup_desc.Dump("Duplicate descriptions in batch");

// Validation 2: Spaces in the code field
var spacesInCode = batch.Where(c=> c.Code.Contains(" "));
spacesInCode.Dump("Spaces in code");

// Validation 3: Code exists in database
var codeExists = from b in batch
  join w in wards on b.Code.Normalize() equals w.Code.Normalize()
  select new {
    Batch_Code = b.Code,
    Batch_Desc = b.Desc,
    Database_Code = w.Code,
    Database_Desc = w.Desc
  };
codeExists.Dump("Code exists in database");

// Validation 4: Description is same as one in database
var descExists = from b in batch
  join w in wards on b.Desc.Normalize() equals w.Desc.Normalize()
  select new {
    Batch_Code = b.Code,
    Batch_Desc = b.Desc,
    Database_Code = w.Code,
    Database_Desc = w.Desc
  };
descExists
  .Where(d=> {
    // Exclude fails from validation 3
    !codeExists.Select(c=> c.Batch_Code).Contains(d.Batch_Code) 
  }) 
  .Dump("Description is same as one in database");

// -----------------------------------------------------------------------

      # Linqpad Extensions Used

      # Normalize Function

      This function is used to strip non-alpha characters from strings to be compared. It follows the same logic used by the loading routine.

      // Normalize function for Trak string compare
public const string alphaNum =
  "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz1234567890";

public static string Normalize(this string inString) 
{
  return inString
    .ToUpper()
    .Trim()
    .RemoveSpecialChars();
}

public static string RemoveSpecialChars(this string s)
{
  var sb = new StringBuilder();
  foreach(var c in s)
  {
    if (alphaNum.Contains(c))
      sb.Append(c);
  }
  return sb.ToString();
}

      # ODBC connection to the target database

      An external DLL is used to provide low level connectivity to the database. The following function encapsulates the logic for running it.

      The parser parameter is a Func<> delegate which returns an object (of any shape), so the function return is cast to List<dynamic> in order to handle many queries against many different tables without explicitly defining the model for each.

      public static List<dynamic> FetchFromDatabase(
  string query,
  Func<List<string>, object> parser,
  string db = "MAIN",  // allow selection of alternate database
  bool debug = false)
{
  var runner = new RunQuery("Dsn=ENYH-" + db);
  runner.Run(query);
  if(debug) DebugRunner(runner, parser);
  var data = runner.Result.Tables[0].AsEnumerable()
    .Select(r=> ConvertToString(r.ItemArray))
    .Select(r=> parser(r))
    .ToList();
  return data;
}

private static dynamic ConvertToString(object[] row)
{
  var list = new List<string>();
  foreach(var item in row) {
    list.Add(item.ToString());
  }
  return list;
}
